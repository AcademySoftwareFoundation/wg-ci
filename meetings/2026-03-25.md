## New items

* GitHub Actions Security Issues
  * Not just theoretical anymore, [automated via AI agents](https://www.stepsecurity.io/blog/hackerbot-claw-github-actions-exploitation)
  * [OpenSSF Bulletin](https://lists.openssf-vuln.org/g/siren/message/6)
  * Do NOT use `pull_request_target:` which allows running builds on PR in context of foreign repo with access to your secrets
    * All uses pruned from projects under AcademySoftwareFoundation GitHub org
    * Not sure about projects under their own org (how can we keep track of those?)
      * Larry: if a project wants a bunch of different repos, and wants to have a third level to hierarchy, they can. So a bunch of repos would end under the ASWF GitHub repo, could be confusing to a new browser. OIIO has a repo for code and one for test images. I can understand that motivation.
  * Do NOT use `${{ ... }}` for shell level expansion of anything you don't control (env vars, branch names, tag names...)
  * Can pin container images with SHA256
  
  ```yaml
  container:
    image: aswf/ci-openexr@sha256:9ddf6fccd32619a23a02db56aa6761e0b87ffce7374fa103768ded0ae09f1120 # 2026.3
  ```

* AI assisted code review tools
  * Any experience to share?
    * Larry: I've poked the button in GitHub to get CoPilot reviews to see what it has to say. 6 months ago it was mostly a miss, it didn't have anything helpful to say, lots of false negatives. Within last couple of months it is helpful more often than it isn't. And easy to ignore it when it's wrong. It's like a human reviewer who is smart but isn't an insider with the project. They might catch something, or may not know where the project is going. Would not object to an automatic review based on recent advances. Don't trust it on strategic issues, but it does find cut-n-paste errors. It shouldn't take the place of a human review, but it is good at augmenting human reviews. If you are doing something idiomatic to your code base, you have to ignore what it suggests when you know better, don't accept blindly what it says. But it is generally helpful.
  * Does the access to Enterprise-level Copilot for repo maintainers extend to [GitHub Copilot Code Review](https://docs.github.com/en/copilot/how-tos/use-copilot-agents/request-a-code-review/use-code-review)
